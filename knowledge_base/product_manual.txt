深度求索 API 产品手册
====================

1. 快速入门
-----------
要使用 DeepSeek API，你需要一个 API 密钥。请访问 https://platform.deepseek.com
注册账号，然后在控制面板中创建 API 密钥。

API 基础地址：https://api.deepseek.com/v1
认证方式：在 Authorization 请求头中携带 Bearer 令牌

示例请求：
  curl https://api.deepseek.com/v1/chat/completions \
    -H "Content-Type: application/json" \
    -H "Authorization: Bearer 你的API密钥" \
    -d '{
      "model": "deepseek-chat",
      "messages": [{"role": "user", "content": "你好"}],
      "stream": false
    }'

2. 可用模型
-----------
- deepseek-chat：通用对话模型，128K 上下文窗口
- deepseek-coder：编程专用模型，128K 上下文窗口
- deepseek-reasoner：增强推理能力，思考时间更长

3. 函数调用
-----------
DeepSeek 支持兼容 OpenAI 格式的函数调用。通过定义工具的名称、
描述和参数模式，模型会智能判断何时调用函数。

工具定义示例：
{
  "type": "function",
  "function": {
    "name": "get_weather",
    "description": "获取指定城市的天气信息",
    "parameters": {
      "type": "object",
      "properties": {
        "city": {"type": "string", "description": "城市名称"}
      },
      "required": ["city"]
    }
  }
}

4. 流式响应
-----------
设置 "stream": true 即可接收服务器发送事件（SSE），实现实时
逐令牌生成。每个数据块包含带有内容的增量数据。

5. 速率限制
-----------
- 免费套餐：100 次请求/小时
- 基础套餐：1000 次请求/小时
- 专业套餐：10000 次请求/小时
- 企业套餐：可定制限制

6. 上下文窗口管理
-----------------
128K 上下文窗口支持超长对话和文档处理。
可使用 last_message_only 参数或手动截断历史记录
以获得最佳性能和成本效率。

7. 错误代码
-----------
- 401：API 密钥无效
- 429：超出速率限制
- 500：服务器内部错误
- 503：服务暂时不可用

8. 最佳实践
-----------
- 使用系统提示词设定智能体行为
- 实现指数退避重试机制
- 缓存高频请求结果
- 使用流式响应提升用户体验
- 通过响应头监控令牌使用量
