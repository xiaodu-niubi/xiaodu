# DeepSeek 技术规格

## 模型架构

### DeepSeek-V2 架构
- **类型**：混合专家（MoE）Transformer
- **总参数量**：2360亿
- **每令牌激活参数**：210亿
- **注意力机制**：多头潜在注意力（MLA）
- **层数**：60层 Transformer
- **隐藏层维度**：5120
- **专家数量**：160个专家，每令牌激活前6个

### 多头潜在注意力（MLA）
MLA 相比标准 MHA 将 KV 缓存大小减少 5-10 倍，实现高效推理。核心创新：
- 低秩 Key-Value 联合压缩
- 注意力计算时的上投影
- 解耦旋转位置编码

### DeepSeekMoE
细粒度专家分割与共享专家隔离：
- 1个共享专家（始终激活）
- 64个路由专家（激活前6个）
- 无辅助损失的负载均衡

## 训练细节

### 预训练
- **数据量**：8.1T 多语言文本令牌
- **硬件**：2048块 NVIDIA H800 GPU
- **精度**：FP8 混合精度
- **上下文长度**：基础4K，通过 YaRN 扩展到128K

### 对齐训练
- 基于150万高质量示例的有监督微调（SFT）
- 基于人类反馈的强化学习（RLHF）
- 群体相对策略优化（GRPO）

## API 规格

- **协议**：HTTPS REST + SSE 流式传输
- **认证**：Bearer 令牌
- **速率限制头**：X-RateLimit-Remaining
- **最大请求大小**：10MB
- **超时时间**：60秒（流式：300秒）

## 部署方案

### 云端 API
- DeepSeek 托管服务
- 按令牌计费
- 自动弹性伸缩

### 自托管部署
- 支持 GPU 的 Docker 容器
- 最低配置：8x A100/H800（80GB）
- 推荐配置：16x H800 用于生产环境
- 支持 vLLM 和 TensorRT-LLM 推理引擎

## 安全保障

- 所有连接使用 TLS 1.3
- API 密钥静态加密存储（AES-256）
- SOC 2 Type II 认证
- GDPR 合规
- 数据在用户所在区域内处理
